# 1. Introduction

This document describes the non-functional requirements of the Event Hall Management System.

These requirements define how the system should perform rather than what the system should do.

---

# 2. Usability

### NFR-001

The application shall be easy to use without technical knowledge.

---

### NFR-002

The main functions shall be accessible within a maximum of three user interactions.

---

### NFR-003

The interface shall be optimized for daily use by the event hall owner.

---

# 3. Performance

### NFR-004

The application shall display calendar information in less than two seconds under normal operating conditions.

---

### NFR-005

Search operations shall return results in less than one second whenever possible.

---

# 4. Compatibility

### NFR-006

The application shall work correctly on modern desktop web browsers.

---

### NFR-007

The application shall be fully responsive and usable on smartphones and tablets.

---

# 5. Reliability

### NFR-008

Reservation information shall be stored permanently after saving.

---

### NFR-009

The system shall prevent accidental data loss during normal operation.

---

# 6. Security

### NFR-010

Access to the application shall require user authentication.

---

### NFR-011

Only authenticated users shall be allowed to access reservation information.

---

# 7. Maintainability

### NFR-012

The application shall follow a modular architecture to simplify future maintenance.

---

### NFR-013

The source code shall follow consistent coding conventions.

---

# 8. Scalability

### NFR-014

The system architecture shall allow future support for multiple event halls.

---

### NFR-015

The database shall be designed to support future features without major structural changes.

---

# 9. Availability

### NFR-016

The application shall be available whenever the owner needs to manage reservations.

---

# 10. Localization

### NFR-017

Dates, times and currency shall follow Mexican conventions.
