# 1. Purpose

The purpose of this project is to replace the current Excel-based reservation management process with a web application that centralizes the daily operations of an event hall.

The application is intended to simplify reservation management, improve organization, reduce manual work and minimize human errors while providing a better experience for the business owner.

---

# 2. Business Overview

The business is an event hall located in Guadalupe, Zacatecas, Mexico.

Customers usually contact the owner through Facebook, Facebook Messenger, WhatsApp or phone calls after finding advertisements in local Facebook groups.

Reservations are currently managed manually using Microsoft Excel.

Although the current workflow has been sufficient for the business, the increasing number of reservations and operational tasks has made manual management less efficient.

---

# 3. Problem Statement

The current workflow depends on multiple disconnected tools.

These include:

- Facebook Groups
- Facebook Messenger
- WhatsApp
- Phone calls
- Microsoft Excel

As a result:

- Reservation information is scattered across different platforms.
- Availability must be checked manually.
- Payments are tracked manually.
- Customer requirements are difficult to organize.
- Repetitive information must be copied manually.
- Important reminders may be forgotten.
- There is no centralized customer history.

The current process works, but it requires constant manual effort and increases the possibility of human error.

---

# 4. Proposed Solution

Develop a web-based management system that centralizes all business operations into a single application.

The system will provide tools for managing:

- Customers
- Reservations
- Calendar
- Payments
- Customer requirements
- Internal reminders
- Daily operations
- Business reports

The application should be accessible from both desktop computers and mobile devices.

---

# 5. Project Goals

The primary goals of this project are:

- Centralize all reservation information.
- Replace the current Excel spreadsheet.
- Reduce repetitive manual work.
- Improve reservation organization.
- Simplify payment tracking.
- Improve event preparation.
- Keep customer information organized.
- Reduce the possibility of forgotten tasks.
- Provide useful business information through reports and dashboards.

---

# 6. Stakeholders

## Primary Stakeholder

Event hall owner.

Responsibilities include:

- Responding to customer inquiries.
- Managing reservations.
- Scheduling hall visits.
- Collecting payments.
- Supervising events.
- Maintaining the venue.

## Secondary Stakeholders

Customers who rent the event hall.

Although customers will not directly use the application during the first version, their information will be managed by the system.

---

# 7. Project Scope

## Version 1 (Minimum Viable Product)

The first version of the system will include:

- Reservation management.
- Customer management.
- Event calendar.
- Payment tracking.
- Customer notes.
- Special customer requirements.
- Internal reminders.
- Dashboard.

## Future Versions

Possible future improvements include:

- Contract generation.
- Event packages.
- Public website.
- Online availability requests.
- Multiple event halls.
- Employee accounts.
- Inventory management.
- Customer portal.
- Automatic notifications.
- Financial reports.

---

# 8. Success Criteria

The project will be considered successful if:

- The Excel spreadsheet is no longer required for daily operations.
- Reservations can be managed entirely through the application.
- Reservation availability can be checked quickly.
- Customer information is centralized.
- Payment tracking becomes simpler.
- Daily event preparation becomes easier.
- The owner spends less time performing repetitive administrative tasks.
